## 検査情報入力機能 要件定義

### 1. 概要

検査情報入力のワークフローを変更し、検査項目の選択を起点とした操作フローに改修する。

---

### 2. 現状フロー（AS-IS）
```
図面画像をタップ → 検査情報入力
```

- 図面上の位置指定が起点となっている
- 検査項目の選択と図面プロットが一体化している

---

### 3. 変更後フロー（TO-BE）
```
検査項目一覧から選択 → 判定結果入力 → [b2/cの場合のみ] 図面プロット → 撮影 → 不具合情報記入
```

| ステップ | 処理内容 | 条件 |
|---------|---------|------|
| 1 | 検査項目一覧を表示 | - |
| 2 | 検査項目を選択 | - |
| 3 | 判定結果を入力（a/b1/b2/c 等） | - |
| 4 | 図面上に位置をプロット | **b2またはcの場合のみ** |
| 5 | 写真撮影 | **b2またはcの場合のみ** |
| 6 | 不具合情報を記入 | **b2またはcの場合のみ** |

---

### 4. 機能要件

#### 4.1 検査項目一覧画面

| 項目 | 内容 |
|------|------|
| 表示内容 | [ーー]ファイルに定義された検査項目を一覧表示 |
| 選択方式 | 単一選択（1項目ずつ検査を実施） |
| 状態表示 | 未検査/検査済/不具合ありの状態を識別可能にする |

#### 4.2 判定結果入力

| 項目 | 内容 |
|------|------|
| 判定区分 | a / b1 / b2 / c（※要確認） |
| 分岐条件 | b2またはcを選択した場合のみ、後続の図面プロット・撮影・不具合記入へ遷移 |

#### 4.3 図面プロット（条件付き）

| 項目 | 内容 |
|------|------|
| 表示条件 | 判定結果がb2またはcの場合のみ |
| 操作 | 図面画像上をタップして不具合位置を指定 |

#### 4.4 写真撮影（条件付き）

| 項目 | 内容 |
|------|------|
| 表示条件 | 判定結果がb2またはcの場合のみ |
| 機能 | カメラ起動、撮影、保存 |

#### 4.5 不具合情報記入（条件付き）

| 項目 | 内容 |
|------|------|
| 表示条件 | 判定結果がb2またはcの場合のみ |
| 入力項目 | （※[ーー]ファイルの定義に準拠） |

---

### 5. 確認事項

| No | 確認内容 |
|----|---------|
| 1 | 判定区分（a/b1/b2/c）の定義・正式名称 |
| 2 | a/b1の場合、判定結果入力後はどこへ遷移するか（一覧に戻る？次の項目へ自動遷移？） |
| 3 | 1つの検査項目に対して複数箇所の不具合をプロット可能か |
| 4 | 写真は必須か任意か、枚数上限はあるか |
| 5 | [ーー]ファイルの正式ファイル名・格納場所 |
